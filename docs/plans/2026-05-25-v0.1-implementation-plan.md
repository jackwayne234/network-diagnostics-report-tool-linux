# Network Troubleshooting App v0.1 Implementation Plan

> **For Hermes:** Use this plan as the task list for the initial local Python/PySide6 implementation.

**Goal:** Build the first working Linux-only, read-only Network Troubleshooting App skeleton with real command adapters and a usable PySide6 tabbed GUI.

**Architecture:** A Python package under `src/net_troubleshooter` separates command adapters from GUI widgets. Backend adapters run allowlisted Linux commands with timeouts and return structured `CommandResult` objects. The GUI uses PySide6 tabs, status cards, expandable command details, and current-session-only results.

**Tech Stack:** Python 3, PySide6, pytest, standard-library `subprocess`, JSON/Markdown export later.

## Phase 1: Project skeleton and backend core

### Task 1: Create Python project structure

**Objective:** Add a normal source-layout Python project that can be installed in editable mode.

**Verification:**
Run `python -m compileall src` and expect success.

### Task 2: Add command result model and safe command runner

**Objective:** Implement allowlisted command execution with timeout handling and structured results.

**Files:**
- Create: `src/net_troubleshooter/core/models.py`
- Create: `src/net_troubleshooter/core/runner.py`
- Test: `tests/test_runner.py`

**Acceptance:**
- Runner accepts command name + list of args, not arbitrary shell strings.
- Runner records command, args, exit code, stdout, stderr, start/end timestamps, duration, timed_out, and missing_tool.
- Missing tools return a skipped-style result instead of crashing.

### Task 3: Add first diagnostic adapters

**Objective:** Add read-only command adapters for dashboard and quick check.

**Files:**
- Create: `src/net_troubleshooter/core/adapters.py`
- Test: `tests/test_adapters.py`

**Adapters:**
- `InterfaceStatusCommand` using `ip addr`
- `RouteTableCommand` using `ip route`
- `DnsStatusCommand` using `resolvectl status` with fallback handling
- `PingCommand` using `ping -c 4 -W 2 TARGET`
- `DnsLookupCommand` using `dig +short TARGET` if available, fallback message if missing
- `HttpCheckCommand` using `curl -I --max-time 8 URL`

### Task 4: Add quick check service

**Objective:** Implement the balanced Quick Check sequence.

**Files:**
- Create: `src/net_troubleshooter/core/quick_check.py`
- Test: `tests/test_quick_check.py`

**Steps:**
1. Detect interface/IP info.
2. Detect default gateway.
3. Ping gateway if found.
4. Ping `1.1.1.1`.
5. Resolve `example.com`.
6. Check HTTPS to `https://example.com`.
